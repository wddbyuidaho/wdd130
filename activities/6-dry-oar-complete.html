This week we will finally finish up the Dry Oar rafting site. To do this you should first make sure that you have made any changes suggested in feedback on your previous assignments, and then create a second page for your site.

In case you have forgotten, here are the requirements for the assignment:

The web site must consist of a main page and at least 1 child page. The web site must be published to the web. Each of the pages of your web site must satisfy the following requirements:

  • Significant content consisting of both text and a minimum of one image. You need to make sure this page has the appropriate content for your audience so that they can understand the meaning and message of this site.
  • Working links to the all of the other pages you have created.
  • Be visually appealing

Activity Instructions

  1. Review Feedback

    For this part of the project, make sure to review any feedback you have received on your DryOar website and make any corrections or enhancements to make sure it looks and functions great. Make sure all images display and all links either work or do nothing. Make sure there are no spelling errors as well.

  2. Create the new HTML file for the child (sub) page

    Choose one of the pages you identified last week and made wireframes for that you will complete as your child-page. A child-page is a page with links back and forth to your main page. You only need to code one child-page for the Dry Oar site.

    Want a shortcut? We know that many of the elements from our home page should also be on your child pages...header, navigation, footer, colors, etc. So use the index.html as a starting point. Make a new file in your editor and save it with a meaningful name (for example if I were making a contact page I would name my new file contact.html)...then copy and paste the code from your index.html into your new file, update the <title>, and delete the content that is specific to the home page and Voila! you have a great start on your new page.

    Note

    You do not want to create a new .css file for this new page. This could create a problem of having pages look different in important ways because the styles in one file were updated and not the other. You should only have one .css file for your entire site. Add any new styles for this new html page to the bottom of your existing .css.

  3. Add new content

    Add the html necessary in your new page to contain the new content.

    Remember that the content I have provided can be found in Ilearn or here: subpage content.

    Pay attention to your wireframe and try to make the HTML represent your layout, ie. create boxes (sections or divs) to represent the rows and columns of your layout and to group related content together. Open your website into your browser to make sure the web page displays correctly. Pay special attention to whether your images are all displaying. Once verified, begin adding the CSS to your site .css file to finish the page.

  4. Submit

    Validate your html and CSS, fix any errors, then submit the URL to your site in ILearn.

What is meant by “visually appealing?”

I wish there was a non-subjective way to measure this... but there isn't. Generally you only lose points for this with things that are obvious. For example, text that is difficult to read because it uses a color to similar to the background, a navigation bar that is directly against a banner or the text of the page rather than having some space around it, use of a color scheme that obviously indicates no thought was put into the color choices (this doesn't mean that you can't use white - it's more stating that if you just threw colors onto the page for the sake of throwing colors onto the page and it is obvious, you'll lose points), etc.

If you simply copy/pasted the text from the content page in most cases it will not be visually appealing. Make sure that you format it to make it more readable! Be generous with <p> tags, padding, margins, and maybe even <ul>s. You will lose points on this if you do not put some thought into the readability of the text.

What is considered “significant content?”

Each page of a website should clearly convey to the user what the page is for. If it is a main/home page, it should provide the user with enough information to know what your site is about, whether it contains the information/products they are looking for, and, thereby, whether they should stick around and keep looking through your website. As another example, if you have a "Contact Us" page, you usually need more than just a contact form/email address. The page should have enough information that the user knows what the contact form/email address is for, what happens when they contact you, what type of information you want the user to provide when contacting you, etc.
